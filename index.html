Stage BUT3 · Terminé

RJ10 Rappel

Application web de relance automatisée pour le renouvellement des certifications professionnelles — réalisée en autonomie lors de mon stage de fin d'études chez RJ10 Formation.

Symfony 7 Vue.js 3 JWT Auth Symfony Mailer Asynchrone Messenger

01 Contexte & Problématique

RJ10 Formation est un organisme spécialisé en sécurité professionnelle basé à Troyes. La gestion manuelle du renouvellement périodique des habilitations (SST, habilitation électrique, etc.) pour des milliers d'employés s'avérait chronophage et propice aux oublis critiques. Mon rôle a été de concevoir de A à Z une application moderne capable d'assurer ce suivi de façon entièrement automatisée.

Aperçu Conceptuel · Dashboard
Certifications en cours de validité 98.2% Conformes
Habilitation Électrique (B2V) · Entreprise AXA J-14 avant relance

02 Fonctionnalités intégrées

  • ✦ Analyse automatique quotidienne des échéances et envoi d'emails responsives.
  • ✦ Espace d'administration complet (CRUD) pour les clients, formateurs et types de certificats.
  • ✦ File d'attente d'envoi gérée par bus asynchrone (Symfony Messenger) pour fiabiliser les emails.
  • ✦ Module d'invitation utilisateur sécurisé par jetons d'authentification uniques.

Données Métiers

  • Langage API : PHP 8.2 / Symfony 7
  • Client lourd : Vue.js 3
  • Base de données : MySQL
  • Authentification : Stateless (JWT)

Choix d'architecture

Afin d'obtenir une réactivité maximale sur le web, l'application est pensée comme une API RESTful stricte. Le front-end en Vue.js consomme les données asynchrones de manière transparente.

Full-Stack · En cours

Noor App

Application modulaire complète organisée en double-dépôts — intégrant une API REST en Node.js (Express) et une application cliente responsive en React.

Node.js (Express) React (Hooks & Context) PostgreSQL JWT Auth

01 Analyse & Choix Techniques

Noor App a été développée avec une philosophie claire : séparer le serveur applicatif de la couche présentation. L'interface React gère la navigation de manière dynamique et sans rechargement, tandis que l'API construite sous Express assure la validation des données d'entrée, les authentifications cryptées par jetons d'accès et la persistance des relations dans PostgreSQL.

Back-end (noor-back)
  • ✓ Routage modulaire et validation des payloads.
  • ✓ Système de chiffrement asymétrique et JWT.
  • ✓ Base relationnelle performante PostgreSQL.
  • ✓ Gestion rigoureuse des erreurs.
Front-end (noor-app)
  • ✓ Interface utilisateur hautement responsive.
  • ✓ Gestion globale de l'état utilisateur via Context.
  • ✓ Routage client protégé basé sur les rôles.
  • ✓ Intégration soignée (Tailwind CSS).

Fiche d'ingénierie

  • Serveur : Node.js / Express
  • Client : React (Axios)
  • Base : PostgreSQL
  • Sécurisation : JWT Token

Contrôle d'accès

L'application implémente des middlewares spécifiques à chaque route de l'API. Les privilèges de rôles permettent une séparation étanche entre comptes administrateurs et comptes clients simples.
